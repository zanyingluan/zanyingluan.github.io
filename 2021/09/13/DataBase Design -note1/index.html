教师：吴毅坚

第一周 9月13日

Lecture1. Introduction

1.Database Management System(DBMS) - 数据库管理系统

一个数据库管理系统包括数据本身与访问数据（access）的程序，用以构造一个高效且便利的数据管理环境（effecient and convenient）

数据库所管理的数据有这些特点：高度有价值，相对来说体量比较大，相互关联的

Q：为何用数据库管理而不用文件系统？

  • Data redundancy and inconsistency 数据冗余与不一致性
  • Diffculty in accessing data 访问数据的复杂性
  • Data isolation 数据之间存在隔离
  • Integrity problems 产生数据完整性的问题
  • Atomicity problems of updates 数据原子性的问题 （一个地方更新了，那么另外一个地方也需要更新）
  • Concurrent access anomalies 并发的问题 （多用户同时使用）
  • Security problems 安全问题

所有以上的问题，都可以使用DBMS解决！

2.Database - 数据库是森莫

数据库包括：

  • Tables 表格
  • Arrributes（Fields）字段
  • Tuples（Records） 元祖
  • Constraints（Logical relationship among data）

数据层抽象：

  • Physical（物理层）：一条记录如何存储的物理实现
  • Logical（逻辑层）：数据库设计者着重需要考虑
  • View（视图层）：提供应用访问，信息隐藏

模式与实例：

  • Schema（模式）：数据库的逻辑结构
    • Physical shcema：数据库在物理层的设计
    • Logical schema ：数据库在逻辑层的设计（重点关注）
    • Physical Data Independence ：在不影响逻辑层的情况下更改物理层
  • Instance（实例）：数据库在某时某刻的实际内容

存储管理（Storage Manager）

  • 提供层与层之间存在的一个接口，用以实现物理数据独立性
  • ss
  • s

语言处理程序（Query Processing）

  1. Parsing and translation
  2. Optimization
  3. Evaluation

事务管理（Transaction Management）

  • Transaction-management component 单事物管理
  • Concurrency-control management 并发访问控制

3.谁使用数据库

管理员职责包括：

  • Schema definition
  • Storage structure and access method definition
  • Schema and physical organization modification
  • Granting user authority to access the database
  • Specifying integrity constraints
  • Acting as liaison with users
  • Monitoring performance and responding to changes in requirements

其他使用者包括专业、业余的各种各样的使用者

4.如何让数据库工作？

DDL（Data Definition Language）数据定义语言 -主要用于数据定义

DML（Data Manipulation Language）数据操纵语言 - 主要进行增删改查

  • 过程式DML （Procedural）将处理数据的过程描述清楚 -算数表达
  • 描述式/非过程式DML（Declarative）（nonprocedural）不用描述过程，只需要描述条件 -sql语句

数据模型（Data Models）

  • 一种抽象描述数据，数据之间关系，数据限制等的工具
  • 关系模型 （涉及到关系代数，SQL语句的数学基础）
  • Entity-Realtionship 数据模型 （实体联系模型）
  • 基于对象的数据模型（OO and object-relational）
  • 半结构化数据模型（XML）

Lecture2. Relation Database

关系（Relation）

  • Is used to refer to a table
  • conceptual representation of a table

属性（Attribute）

  • 表的一列
  • 需要有名字，需要原子化，可以取空值

元组（Tuple）

  • 表的一行

Relation Scheama：关系的结构 即表结构

键（Key）

  • 超键（Superkey）：属性的集合，足够识别某一特地的元组的一些值的集合
  • 候选键（Candidate key）：超键，且是最小识别的超键，再少一个属性都不是超键
  • 主键（Primary key）：一个已经被选择的候选键
  • 外键（Foreign key）：一组属性，是另外一个关系里的主键，可以说是指向某个关系的外键
